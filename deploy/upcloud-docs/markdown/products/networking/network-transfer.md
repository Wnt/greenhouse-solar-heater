# Network Transfer

UpCloud's networking is provided as zero-cost for most use cases. [Read more about zero-cost egress](https://upcloud.com/products/zero-cost-egress/).

Public egress transfer is included in all Cloud Server plans. [Fair Transfer Policy](https://upcloud.com/fair-transfer-policy/) applies for high bandwidth scenarios. The Fair Transfer Limit is calculated based on your cloud resource usage.

Public ingress transfer and private network transfer in both the Utility network and SDN Private Networks are included in all Cloud Servers.

| Transfer type | Ingress | Egress |
| --- | --- | --- |
| Public networks | Included | Included within the Fair Transfer Limit |
| Utility network | Included | Included |
| SDN Private Networks | Included | Included |

The Fair Transfer Limit is global and shared between all data centres.

A Cloud Server contributes to the Fair Transfer Limit for the entire duration it is deployed, from the moment it is created until it is deleted.

Services created or terminated mid-month will accrue a proportional amount of transfer.

When the Fair Transfer Limit is reached, a warning email is sent, after which we may throttle the bandwidth on the cloud services down to 100 Mbit/s.

Network transfer statistics can be viewed in the [UpCloud Control Panel](https://hub.upcloud.com/usage/network-transfer).
