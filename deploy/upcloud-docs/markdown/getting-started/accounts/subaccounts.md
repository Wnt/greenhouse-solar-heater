# Subaccounts

The main user of their UpCloud account is able to create [additional subaccounts](/docs/getting-started/accounts.md) to give access to their team members without sharing their own login credentials.

The account holder can set the user permissions for their workspace members by limiting members’ access to a specific server, storage and network resources as well as by using the tagging feature.

It’s also recommended to set up a [separate subaccount for each API user](/docs/guides/getting-started-upcloud-api.md) and integrations with 3rd party services.

### Billing contact

A subaccount can be set as the primary billing contact. This means that the subaccount will receive all billing-related emails & notifications. The billing contact's contact details such as address & phone number will be used in invoices and when adding new payment methods. The primary billing contact can also manage the billing details, view history and payment methods for the account.
