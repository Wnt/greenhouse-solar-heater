# Quickstart: Deploy PoC Web UI to Cloud

**Feature Branch**: `001-deploy-web-ui-cloud`
**Date**: 2026-03-20

## Prerequisites

- Node.js 20 LTS
- Docker and Docker Compose v2
- Terraform 1.5+
- UpCloud account with API credentials
- Cloudflare account (free tier) with a domain
- UniFi gateway with Network 8.1+ (for WireGuard VPN Client)

## Local Development

### Run the PoC server locally (no auth, no VPN)

```bash
cd poc
node server.js
# Open http://localhost:3000
```

### Run with auth enabled (for testing passkey flow)

```bash
cd poc
AUTH_ENABLED=true RPID=localhost ORIGIN=http://localhost:3000 node server.js
# Open http://localhost:3000
# Complete passkey registration within 30-minute window
```

### Run tests

```bash
npm test              # All tests (unit + simulation + e2e)
npm run test:unit     # Unit tests only (fast)
npm run test:e2e      # Playwright e2e tests
```

## Docker (Local)

### Build and run locally

```bash
cd deploy/docker
docker compose build
docker compose up
# Caddy proxies https://localhost → app:3000
```

## Production Deployment

### 1. Provision infrastructure with Terraform

```bash
cd deploy/terraform

# Set credentials
export UPCLOUD_USERNAME="your-username"
export UPCLOUD_PASSWORD="your-password"
export CLOUDFLARE_API_TOKEN="your-token"

# Configure variables
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars: domain, SSH key, Cloudflare zone

terraform init
terraform plan
terraform apply
```

This creates:
- UpCloud server (1xCPU-2GB, fi-hel1, Ubuntu 24.04)
- Firewall rules (SSH, HTTPS, WireGuard UDP 51820)
- Cloudflare DNS A record
- Docker + WireGuard installed via cloud-init

### 2. Configure WireGuard VPN

After Terraform completes:

```bash
# SSH into the server
ssh root@$(terraform output -raw server_ip)

# WireGuard keys are generated by cloud-init
cat /etc/wireguard/public.key
# Copy this public key for UniFi configuration
```

On UniFi Controller:
1. Settings > VPN > VPN Client > Create New (WireGuard)
2. Enter cloud server public key and endpoint (`server_ip:51820`)
3. Configure which networks to route through VPN (the LAN where Shelly devices reside)
4. Note the UniFi public key displayed

Back on the cloud server:
```bash
# Edit /etc/wireguard/wg0.conf — add UniFi's public key as peer
# Then: systemctl restart wg-quick@wg0
```

### 3. Deploy the application

The first deploy is manual (subsequent deploys are automatic via GitHub Actions):

```bash
ssh deploy@$(terraform output -raw server_ip)
cd /opt/app
docker compose pull
docker compose up -d
```

### 4. Register passkey

1. Navigate to `https://your-domain.com` within 30 minutes of first deployment
2. You'll see the registration page
3. Follow the passkey enrollment flow
4. After registration, the setup window closes permanently

## Continuous Deployment

After initial setup, merging to `main` with passing tests triggers automatic deployment:

1. GitHub Actions runs tests
2. Builds Docker image and pushes to GHCR
3. SSHs into UpCloud server and runs `docker compose pull && up -d --wait`

### Required GitHub Secrets

| Secret | Value |
|--------|-------|
| `DEPLOY_SSH_KEY` | Ed25519 private key for the `deploy` user |
| `DEPLOY_HOST` | UpCloud server IP or domain |
| `DEPLOY_USER` | `deploy` |

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `AUTH_ENABLED` | `true` | Enable/disable passkey authentication |
| `RPID` | (required) | WebAuthn Relying Party ID (your domain) |
| `ORIGIN` | (required) | Full origin URL (e.g., `https://monitor.example.com`) |
| `SESSION_SECRET` | (required) | Random secret for HMAC session signing |
| `PORT` | `3000` | Server listening port |
| `CREDENTIALS_PATH` | `./auth/credentials.json` | Path to credential store |
| `SETUP_WINDOW_MINUTES` | `30` | Initial registration window duration |
