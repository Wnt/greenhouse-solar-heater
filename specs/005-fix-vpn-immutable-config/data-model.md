# Data Model: Mutable Server Configuration

**Feature**: 005-fix-vpn-immutable-config
**Date**: 2026-03-21

## Entities

### Bootstrap Secrets (`.env.secrets`)

Written once by cloud-init during server provisioning. Never modified after initial creation.

| Field | Description | Source | Mutable |
|-------|-------------|--------|---------|
| SESSION_SECRET | HMAC key for session cookies | Terraform variable | No |
| S3_ENDPOINT | Object storage endpoint URL | Terraform-provisioned resource | No |
| S3_BUCKET | Bucket name | Terraform-provisioned resource | No |
| S3_ACCESS_KEY_ID | S3 access key | Terraform-provisioned resource | No |
| S3_SECRET_ACCESS_KEY | S3 secret key | Terraform-provisioned resource | No |
| S3_REGION | S3 region identifier | Terraform variable | No |

**Lifecycle**: Created at server provision. Destroyed only when server is destroyed. Survives deployer cycles, container restarts, and image updates.

### Service Configuration (`config.env`)

Baked into the deployer image. Updated on every git push via CD pipeline.

| Field | Description | Source | Default |
|-------|-------------|--------|---------|
| PORT | App listening port | Static | 3000 |
| AUTH_ENABLED | Enable passkey authentication | Static | true |
| RPID | WebAuthn Relying Party ID (domain) | Repo config | (project domain) |
| ORIGIN | WebAuthn origin URL | Repo config | https://(project domain) |
| DOMAIN | Caddy reverse proxy domain | Repo config | (project domain) |
| GITHUB_REPO | Container image registry path | CI build arg | (auto-detected) |
| VPN_CHECK_HOST | Host to verify VPN connectivity | Repo config | (empty) |
| VPN_CONFIG_KEY | S3 key for WireGuard config | Static | wg0.conf |
| SETUP_WINDOW_MINUTES | Initial registration window | Static | 30 |
| NODE_ENV | Node.js environment | Static | production |
| COMPOSE_PROFILES | Docker Compose profiles to activate | Repo config | (empty) |

**Lifecycle**: Rebuilt with every deployer image. Applied to server on each deployer cycle (~5 min). Old values replaced atomically.

### Merged Environment (`.env`)

Runtime file consumed by Docker Compose. Generated by deployer merge step.

**Merge rule**: All keys from `.env.secrets` + all keys from `config.env`. If a key exists in both, `.env.secrets` wins (secrets are authoritative).

**Lifecycle**: Regenerated on every deployer cycle. Previous version overwritten atomically.

## State Transitions

```
Server Provision (terraform apply - first time)
  → cloud-init writes .env.secrets
  → deployer timer starts
  → first deployer cycle: copies config.env, merges → .env, starts services

Config Change (git push)
  → CD builds new deployer image
  → deployer timer pulls new image
  → deployer copies new config.env, merges → .env, restarts services

VPN Enable (git push + terraform apply)
  → git push: config.env with COMPOSE_PROFILES=vpn → deployer starts wireguard
  → terraform apply: firewall rule opens port 51820 → external traffic reaches wireguard

Secret Rotation (terraform apply - server recreation)
  → Server destroyed and recreated (expected, acceptable)
  → cloud-init writes new .env.secrets
  → deployer applies config.env from latest image
```
