# Data Model: Switch to OpenVPN

## Entities

### VPN Server Configuration (`openvpn.conf`)

Single text file containing the complete OpenVPN server configuration with embedded static key.

| Field | Description |
|-------|-------------|
| Protocol | UDP (fixed) |
| Port | 1194 (configurable) |
| Device type | tun (routed mode) |
| Server tunnel IP | Point-to-point address, server end (e.g., 10.10.10.1) |
| Client tunnel IP | Point-to-point address, client end (e.g., 10.10.10.2) |
| Remote network | Client-side LAN to route through tunnel (e.g., 192.168.1.0/24) |
| Static key | 2048-bit OpenVPN static key, embedded inline via `<secret>` tag |
| Keepalive | Ping interval and timeout (e.g., 10s ping, 60s timeout) |

**Persistence**: S3 object storage (key: `openvpn.conf`, configurable via `VPN_CONFIG_KEY` env var)

**Lifecycle**:
1. Generated by `deploy/openvpn/setup.sh` during initial setup
2. Placed at `/opt/app/openvpn.conf` on the server
3. Uploaded to S3 on first deployer run (bootstrap)
4. Downloaded from S3 on subsequent deployer runs (recovery)

### Firewall Rule

| Field | Description |
|-------|-------------|
| Direction | Inbound |
| Protocol | UDP |
| Port | 1194 |
| Condition | `enable_vpn = true` (Terraform variable) |

### Environment Variables

| Variable | Source | Description |
|----------|--------|-------------|
| `VPN_CONFIG_KEY` | config.env (mutable) | S3 object key for VPN config (default: `openvpn.conf`) |
| `VPN_CHECK_HOST` | config.env (mutable) | Home LAN IP:port for health check (unchanged) |
| `COMPOSE_PROFILES` | .env (server) | Must include `vpn` to enable OpenVPN container |

## State Transitions

### VPN Container Lifecycle

```
[Not configured] → (setup.sh generates config) → [Config ready, VPN disabled]
[Config ready, VPN disabled] → (set COMPOSE_PROFILES=vpn, enable_vpn=true) → [VPN enabled, waiting for client]
[VPN enabled, waiting for client] → (UniFi connects) → [Tunnel active]
[Tunnel active] → (UniFi disconnects / network issue) → [Tunnel reconnecting]
[Tunnel reconnecting] → (keepalive restores) → [Tunnel active]
```

### S3 Config Persistence Lifecycle

```
[No S3 config] → (deployer uploads local config) → [S3 config exists]
[S3 config exists] → (server recreated, deployer downloads) → [Local config restored]
[S3 config exists] → (deployer finds local already exists) → [Skip download, skip upload]
```
